شرح كود نموذج تسجيل الدروس (LessonEnrollment)

هذا الكلاس (Model) يمثل "طلب التسجيل" الذي يرسله الطالب للمعلم للانضمام إلى درس معين. هو الرابط الأساسي بين الطالب والدرس في قاعدة البيانات.

1. حالات الطلب الثابتة (Status Constants)

يحتوي الكود على مجموعة من القيم الثابتة التي تحدد حالة الطلب في أي لحظة. استخدام الكلمات الثابتة يمنع الأخطاء الإملائية في الكود ويجعله أكثر تنظيماً:
  • STATUS_PENDING: الطلب قيد الانتظار (بانتظار موافقة المعلم).
  • STATUS_APPROVED: تم قبول الطالب في الدرس.
  • STATUS_REJECTED: تم رفض الطلب من قبل المعلم.
  • STATUS_CANCELLED: تم إلغاء الطلب (من قبل الطالب أو المعلم).

public static final String STATUS_PENDING = "pending";
public static final String STATUS_APPROVED = "approved";
public static final String STATUS_REJECTED = "rejected";
public static final String STATUS_CANCELLED = "cancelled";

2. الحقول الأساسية (Fields)

يمثل كل حقل معلومة ضرورية لإتمام عملية التسجيل:
  • id: المعرف الفريد لهذا التسجيل في قاعدة البيانات.
  • lessonId: معرف الدرس الذي يريد الطالب الانضمام إليه.
  • studentId: معرف الطالب الذي أرسل الطلب.
  • status: الحالة الحالية للطلب (تأخذ إحدى القيم الثابتة أعلاه).

public String id;
public String lessonId;
public String studentId;
public String status;

3. المنشئ الافتراضي (Constructor)

عند إنشاء أي طلب تسجيل جديد، يتم ضبط الحالة تلقائياً لتكون "pending" (قيد الانتظار) كقيمة افتراضية حتى يقوم المعلم بتغييرها لاحقاً. 

public LessonEnrollment() {
    this.status = STATUS_PENDING; // الحالة الافتراضية هي قيد الانتظار
}
