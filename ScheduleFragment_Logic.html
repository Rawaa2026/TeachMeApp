شرح كود شاشة الجدول الزمني (ScheduleFragment)

هذه الشاشة مسؤولة عن عرض "الجدول الزمني" الخاص بالمستخدم، حيث تعرض الدروس المعتمدة (المقبولة) سواء كان المستخدم معلماً أو طالباً.

1. جلب بيانات الجدول (load)

تقوم دالة load() بتحديد نوع المستخدم الحالي من الجلسة (Session)، وبناءً عليه تستدعي الدالة المناسبة من المستودع (Repository):
  • إذا كان معلماً: تجلب الدروس التي يدرسها وتم قبول طلاب فيها.
  • إذا كان طالباً: تجلب الدروس التي سجل فيها وتم قبوله.

private void load() {
    User u = session.getCurrentUser();
    progress.setVisibility(View.VISIBLE);
    if (u.isTeacher()) {
        repo.loadTeacherSchedule(u.id, result -> {
            ArrayList<Lesson> list = result.data;
            bindList(list, false); // false تعني عدم الحاجة لجلب أسماء مدرسين لأنني المدرس
        });
    } else {
        repo.loadStudentSchedule(u.id, result -> {
            ArrayList<Lesson> list = result.data;
            bindList(list, true); // true تعني جلب أسماء المدرسين لكل درس
        });
    }
}

2. ربط البيانات ومعالجة المدرسين (bindList)

عندما يكون المستخدم طالباً، نحتاج لعرض اسم المعلم ورقم هاتفه. بما أن كائن الدرس يحتوي فقط على معرف المعلم (teacherId)، نستخدم خيطاً (Thread) خلفياً لجلب هذه البيانات لكل درس لضمان عدم بطء الواجهة. 

private void bindList(ArrayList<Lesson> lessons, boolean showTeacher) {
    items.clear();
    items.addAll(lessons);
    if (showTeacher && !items.isEmpty()) {
        new Thread(() -> {
            // جلب أسماء وأرقام هواتف المدرسين من UserRepository في الخلفية
            for (Lesson l : lessonsSnapshot) {
                User t = ur.getUserById(l.teacherId);
                teacherNames.put(l.teacherId, t.getDisplayName());
                teacherPhones.put(l.teacherId, t.phoneNumber);
            }
            // تحديث الواجهة بعد انتهاء الجلب
            root.post(() -> adapter.notifyDataSetChanged());
        }).start();
    } else {
        adapter.notifyDataSetChanged();
    }
}

3. منطق الإلغاء (Cancel Logic)

تحتوي الشاشة على دوال للإلغاء تختلف باختلاف الدور:
  • cancelLesson: يستخدمها المعلم لإلغاء الحصة بالكامل.
  • cancelEnrollment: يستخدمها الطالب لإلغاء تسجيله في الحصة.
يتم التحقق أولاً من أن وقت الدرس لم يبدأ بعد باستخدام canCancelBeforeLessonStart. 

private void cancelLesson(Lesson lesson) {
    repo.cancelLessonByTeacher(lesson.id, u.id, r -> {
        Toast.makeText(getContext(), "تم إلغاء الدرس بنجاح", Toast.LENGTH_LONG).show();
        load(); // إعادة تحميل الجدول بعد الإلغاء
    });
}

4. التحكم في شكل البطاقة (onBindViewHolder)

يتم تخصيص شكل البطاقة ديناميكياً:
  • إذا كانت الحصة "اليوم"، يظهر شعار (Today Badge).
  • إذا كانت الحصة قديمة (Past)، يتم وضع خط على النصوص (Strike Through) لتبدو منتهية.
  • زر الإلغاء يظهر فقط إذا كانت الحصة في المستقبل ولم تبدأ بعد.
