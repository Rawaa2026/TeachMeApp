شرح كود شاشة الملف الشخصي (ProfileFragment)

هذه الشاشة مسؤولة عن عرض بيانات المستخدم الحالية (سواء كان معلماً أو طالباً)، وتسمح له بتغيير صورته الشخصية، وتعديل بياناته، وحساب إحصائيات الدروس المالية.

1. جلب وتحديث بيانات المستخدم (setupUserProfile)

هذه الدالة تقوم بعملين هامين:
  • أولاً: تعرض البيانات المحفوظة محلياً فوراً (لسرعة الاستجابة).
  • ثانياً: تفتح "خيطاً" (Thread) في الخلفية لجلب أحدث نسخة من بيانات المستخدم من قاعدة البيانات (Appwrite) للتأكد من أنها محدثة.

private void setupUserProfile() {
    currentUser = sessionManager.getUserSession(); // جلب البيانات من الهاتف
    displayUserData(); // عرضها فوراً

    new Thread(() -> {
        // جلب البيانات المحدثة من السيرفر باستخدام المعرف (ID)
        User updatedUser = userRepository.getUserById(currentUser.id);
        requireActivity().runOnUiThread(() -> {
            if (updatedUser != null) {
                currentUser = updatedUser;
                sessionManager.saveUserSession(currentUser); // حفظ التحديث
                displayUserData(); // تحديث الواجهة
            }
        });
    }).start();
}

2. عرض البيانات حسب الدور (displayUserData)

يستخدم الكود منطقاً ذكياً لإظهار وإخفاء الحقول بناءً على دور المستخدم:
  • إذا كان معلماً: يظهر التخصص، سنوات الخبرة، وسعر الساعة.
  • إذا كان طالباً: يظهر تاريخ الميلاد، الصف الدراسي، واسم المدرسة.
كما يتم تغيير الألوان لتناسب هوية الدور (معلم أو طالب). 

if (currentUser.isTeacher()) {
    teacherExtraBlock.setVisibility(View.VISIBLE); // إظهار حقول المعلم
    studentExtraBlock.setVisibility(View.GONE);    // إخفاء حقول الطالب
    profileTeacherSubjects.setText(currentUser.specialization);
} else {
    teacherExtraBlock.setVisibility(View.GONE);
    studentExtraBlock.setVisibility(View.VISIBLE);
    profileStudentGrade.setText(currentUser.grade);
}

3. إدارة الصورة الشخصية (Camera & Gallery)

يحتوي الكود على منطق متكامل للتعامل مع الصور:
  • استخدام ActivityResultLauncher لفتح الاستوديو أو الكاميرا.
  • حفظ الصورة في ذاكرة الهاتف لسرعة العرض لاحقاً.
  • رفع الصورة إلى سيرفر Appwrite وتحديث رابطها في قاعدة البيانات.

private void applyPickedPhoto(Uri uri) {
    // 1. نسخ الصورة للمجلد المحلي للتطبيق
    ProfilePhotoStore.copyFromUri(requireContext(), currentUser.id, uri);

    // 2. تحديث الواجهة فوراً بالصورة الجديدة
    Glide.with(this).load(uri).into(profilePhoto);

    // 3. رفع الصورة للسيرفر وتحديث بيانات المستخدم
    userRepository.updateUser(currentUser, uri, result -> {
        // نجاح الرفع...
    });
}

4. إحصائيات الدروس (loadLessonCostSummary)

دالة برمجية تقوم بحساب المبالغ المالية:
  • للمعلم: تحسب إجمالي أرباحه من الدروس المنشورة اليوم وإجمالي الأرباح الكلية.
  • للطالب: تحسب تكلفة الدروس التي سجل فيها اليوم والإجمالي.
تستخدم هذه الدالة كلاس مساعد يسمى LessonCostHelper للقيام بالعمليات الحسابية.

5. تسجيل الخروج (Logout)

عند الضغط على خروج، يتم مسح "الجلسة" (Session) من ذاكرة الهاتف، وإعادة تشغيل النشاط الرئيسي (MainActivity) الذي سيكتشف عدم وجود مستخدم ويوجهه تلقائياً لشاشة تسجيل الدخول.
تم استخراج هذا التوثيق البرمجي لملف الملف الشخصي في مشروع TeachMeApp.
