شرح كود الشاشة الرئيسية (MainActivity)

تعتبر MainActivity هي الواجهة الرئيسية والمنسق الأساسي للتطبيق، حيث تتحكم في عرض واجهة تسجيل الدخول أو واجهة التطبيق الرئيسية بناءً على حالة المستخدم.

1. البداية والتحقق (onCreate)

عند بدء النشاط، يتم القيام بعدة أمور:
  • تطبيق "الثيم" المناسب بناءً على دور المستخدم (معلم/طالب) عبر RoleThemeHelper.
  • التحقق من حالة تسجيل الدخول عبر SessionManager.
  • إذا كان المستخدم مسجلاً، يتم استدعاء showMainContent()، وإلا يتم استدعاء showAuth() لعرض صفحة الدخول.

protected void onCreate(Bundle savedInstanceState) {
    RoleThemeHelper.applyForLoggedInUser(this); // تطبيق الألوان حسب الدور
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    sessionManager = new SessionManager(this);

    if (sessionManager.isLoggedIn()) {
        showMainContent(); // عرض محتوى التطبيق
    } else {
        showAuth(); // عرض واجهة الدخول
    }
}

2. واجهة تسجيل الدخول (showAuth)

تقوم هذه الدالة "بنفخ" (Inflate) واجهة تسجيل الدخول داخل الإطار الرئيسي (content_frame).
  • تقوم بإخفاء شريط التنقل السفلي.
  • تربط أزرار الدخول والتسجيل بالعمليات البرمجية.
  • عند نجاح الدخول، يتم حفظ الجلسة وإعادة تشغيل النشاط لتحديث الواجهة بالكامل.

private void showAuth() {
    bottomNav.setVisibility(View.GONE); // إخفاء القائمة السفلية
    authView = getLayoutInflater().inflate(R.layout.layout_auth, contentFrame, false);
    contentFrame.addView(authView);

    loginBtn.setOnClickListener(v -> {
        // تنفيذ عملية تسجيل الدخول عبر AuthRepository
        authRepository.login(emailTxt, passwordTxt, result -> {
            if (result.success) {
                sessionManager.saveUserSession(user);
                recreate(); // إعادة تشغيل لتطبيق الثيم والواجهة الجديدة
            }
        });
    });
}

3. واجهة التطبيق الرئيسية (showMainContent)

بعد تسجيل الدخول، يتم عرض الواجهة التي تحتوي على:
  • شريط التنقل السفلي (BottomNavigationView): يتم تغيير أيقونات القائمة ديناميكياً؛ فإذا كان المستخدم معلماً تظهر خيارات المعلم، وإذا كان طالباً تظهر خيارات الطالب.
  • عرض الـ Fragments: يتم تحميل HomeFragment كواجهة افتراضية.

private void showMainContent() {
    bottomNav.setVisibility(View.VISIBLE);
    User sessionUser = sessionManager.getCurrentUser();

    // تغيير القائمة حسب الدور
    if (sessionUser != null && sessionUser.isTeacher()) {
        bottomNav.inflateMenu(R.menu.bottom_nav_teacher);
    } else {
        bottomNav.inflateMenu(R.menu.bottom_nav_student);
    }

    showFragment(new HomeFragment()); // عرض الشاشة الرئيسية الافتراضية
}

4. التنقل بين الشاشات (Navigation)

يتم الاستماع لضغطات المستخدم على القائمة السفلية، وبناءً على العنصر المختار يتم تغيير "العنوان" في الأعلى وتبديل "الـ Fragment" المعروض في منتصف الشاشة. 

bottomNav.setOnItemSelectedListener(item -> {
    int id = item.getItemId();
    if (id == R.id.nav_home) {
        setToolbarTitle(R.string.nav_home);
        showFragment(new HomeFragment());
    } else if (id == R.id.nav_teacher_hub) {
        showFragment(new TeacherHubFragment());
    }
    // ... بقية الخيارات
    return true;
});

5. دالة تبديل الشاشات (showFragment)

دالة مساعدة تستخدم FragmentManager لتبديل المحتوى داخل content_frame بسلاسة. 

private void showFragment(Fragment fragment) {
    getSupportFragmentManager().beginTransaction()
            .replace(R.id.content_frame, fragment)
            .commit();
}
