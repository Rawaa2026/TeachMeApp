شرح كود شاشة تعديل الملف الشخصي (ProfileEditActivity)

هذه الشاشة تسمح للمستخدمين (سواء كانوا معلمين أو طلاباً) بتعديل بياناتهم الشخصية، تغيير صورهم، وتحديث تخصصاتهم أو صفوفهم الدراسية.

1. إدارة الصور (الكاميرا والمعرض)

تستخدم الشاشة نظام ActivityResultLauncher الحديث لاختيار الصور. عند اختيار صورة من المعرض أو التقاطها، يتم حفظ نسخة منها محلياً في مجلد التطبيق باستخدام ProfilePhotoStore لضمان سرعتها الدائمة، ثم يتم عرضها باستخدام مكتبة Glide. 

// لاقط الصور من الاستوديو
private final ActivityResultLauncher<String> pickImage =
    registerForActivityResult(new ActivityResultContracts.GetContent(), uri -> {
        if (uri != null) {
            // حفظ الصورة محلياً وتحديث الواجهة
            ProfilePhotoStore.copyFromUri(this, currentUser.id, uri);
            Glide.with(this).load(uri).centerCrop().into(photoView);
        }
    });

2. تخصيص الواجهة حسب الدور (معلم/طالب)

بناءً على نوع حساب المستخدم المسجل في الجلسة (Session)، يقوم الكود بإظهار أو إخفاء حقول معينة:
  • للمعلم: تظهر حقول التخصص، سنوات الخبرة، وسعر الحصة.
  • للطلاب: تظهر حقول تاريخ الميلاد (مع منقيات التاريخ)، الصف الدراسي، واسم المدرسة.

private void loadUserData() {
    currentUser = sessionManager.getUserSession();
    if (currentUser.isTeacher()) {
        teacherFields.setVisibility(View.VISIBLE); // إظهار حقول المعلم
        studentFields.setVisibility(View.GONE);    // إخفاء حقول الطالب
        setupExperienceYearsSpinner(); // تهيئة قائمة سنوات الخبرة
    } else {
        teacherFields.setVisibility(View.GONE);
        studentFields.setVisibility(View.VISIBLE);
        setupStudentBirthPickers(); // تهيئة قوائم تاريخ الميلاد
    }
}

3. اختيار التخصصات المتعددة (لللمعلمين)

يوفر الكود واجهة حوار (Dialog) تسمح للمعلم باختيار أكثر من مادة دراسية يتقنها. بعد الاختيار، يتم دمج هذه المواد في نص واحد مفصول بأسطر وحفظها. 

private void showTeacherSubjectsPickDialog() {
    new MaterialAlertDialogBuilder(this)
        .setTitle("اختر التخصصات")
        .setMultiChoiceItems(subjectLabels, state, (dialog, which, isChecked) -> {
            state[which] = isChecked;
        })
        .setPositiveButton("موافق", (d, w) -> {
            // دمج المواد المختارة وتحديث الملخص على الشاشة
            refreshTeacherSubjectsSummary();
        })
        .show();
}

4. منطق حفظ البيانات والتحقق

دالة saveProfile() هي المسؤولة عن الجمع والتحقق النهائي:
  • التحقق من أن الاسم والهاتف ليسوا فارغين.
  • التحقق من أن عمر الطالب لا يقل عن 7 سنوات.
  • استدعاء userRepository.updateUser لإرسال البيانات المحدثة والصورة الجديدة إلى السيرفر.
  • تحديث بيانات "الجلسة" (Session) محلياً ليعكس التغييرات فوراً في بقية التطبيق.

private void saveProfile() {
    // 1. جمع البيانات من الواجهة
    sessionUser.fullName = fullNameInput.getText().toString();

    // 2. استدعاء المستودع للتحديث في السحاب
    userRepository.updateUser(sessionUser, photoUri, result -> {
        if (result.success) {
            // 3. حفظ النسخة الجديدة في الجلسة المحلية
            sessionManager.saveUserSession(sessionUser);
            Toast.makeText(this, "تم التحديث بنجاح", Toast.LENGTH_SHORT).show();
            finish();
        }
    });
}
