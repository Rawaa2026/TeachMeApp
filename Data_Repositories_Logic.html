1. مستودع المصادقة (AuthRepository)

هذا الكلاس هو الواجهة الأساسية لعمليات الدخول والتسجيل. وظيفته الأساسية هي تلقي الطلبات من واجهات المستخدم (مثل شاشة تسجيل الدخول) وتمريرها إلى UserRepository، مع ضمان عودة النتائج إلى "الخيط الرئيسي" (Main Thread) لتحديث الواجهة بأمان. 

public class AuthRepository {
    // تسجيل الدخول بالبريد وكلمة المرور
    public void login(String email, String password, AuthCallback callback) {
        userRepository.loginUser(email, password, result -> {
            // إرجاع النتيجة للواجهة بشكل آمن
            mainHandler.post(() -> callback.onResult(result));
        });
    }

    // تسجيل حساب جديد
    public void register(String role, String fullName, ...) {
        User user = new User(email, password, fullName, role);
        userRepository.registerUser(user, photoUri, result -> {
            mainHandler.post(() -> callback.onResult(result));
        });
    }
}

2. مستودع المستخدمين (UserRepository)

هذا هو المحرك المسؤول عن إدارة بيانات المستخدمين في قاعدة بيانات Appwrite. يقوم بالمهام التالية:
  • إنشاء حسابات المستخدمين في نظام Appwrite Account.
  • حفظ بيانات المستخدم التفصيلية في جدول (Collection) يسمى users.
  • معالجة ورفع الصور الشخصية وتحديث روابطها.
  • جلب بيانات أي مستخدم عن طريق معرفه (ID).

public void registerUser(User user, Uri photoUri, AuthCallback callback) {
    ioExecutor.execute(() -> {
        // 1. إنشاء الحساب في Appwrite Authentication
        dal.createDefaultUser(user.email, user.password, ...);

        // 2. حفظ البيانات الإضافية في جدول الـ Users
        persistUserRow(sessionUser);
    });
}

public void updateUser(User user, Uri photoUri, AuthCallback callback) {
    // منطق تحديث البيانات ورفع الصورة الجديدة
    if (photoUri != null) {
        dal.uploadFile(rawBytes, fileName, ...);
    }
    dal.patchDocumentData("users", null, user.id, dataMap);
}

3. مستودع الدروس (LessonRepository)

مسؤول عن إدارة دورة حياة "الدرس" بالكامل. يتعامل مع جدولين في قاعدة البيانات: lessons و lesson_enrollments.
  • نشر الدروس: إضافة درس جديد مع الموعد والسعر.
  • الطلبات: معالجة طلبات الطلاب للانضمام (قيد الانتظار، قبول، رفض).
  • الجدول الزمني: جلب قائمة الدروس "المقبولة" فقط لكل مستخدم ليعرضها في جدول مواعيده.
  • الإلغاء: إلغاء الحصة بالكامل من قبل المعلم أو انسحاب الطالب.

public void publishLesson(String teacherId, String subject, ...) {
    // التأكد من أن وقت الدرس في المستقبل
    if (!isStartAtInFuture(startAtIso)) return;

    // حفظ الدرس في قاعدة البيانات
    dal.createDocumentWithData("lessons", null, docId, data);
}

public void setEnrollmentStatus(String enrollmentId, String newStatus, ...) {
    // تغيير حالة الطلب (قبول أو رفض)
    dal.patchDocumentData("lesson_enrollments", null, enrollmentId, patch);
}

4. مستودع الملف الشخصي (ProfileRepository)

وحدة متخصصة في إدارة الملف الشخصي (Profile) بشكل مستقل. تُستخدم غالباً لتحديث البيانات الفرعية والصور الشخصية بشكل منفصل عن بيانات الحساب الأساسية. 

public void getProfile(String userId, ProfileCallback callback) {
    new Thread(() -> {
        // جلب وثيقة الملف الشخصي بالمعرف
        dal.getDataById("profiles", userId, null, UserProfile.class);
    }).start();
}

public void updateProfile(UserProfile profile, Uri photoUri, ...) {
    // رفع الصورة أولاً إذا وجدت، ثم تحديث البيانات
    if (photoUri != null) {
        dal.uploadFile(...);
    }
    dal.updateData(profile, "profiles", profile.userId, null);
}
