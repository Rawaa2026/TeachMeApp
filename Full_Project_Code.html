1. الاتصال بقاعدة البيانات (DALAppWriteConnection)

2. الشاشة الرئيسية (MainActivity)

MainActivity.java
تتحكم هذه الشاشة في تدفق التطبيق. تقوم بالتحقق من حالة تسجيل الدخول عند بدء التشغيل. إذا كان المستخدم ضيفاً، تعرض له واجهة الدخول، وإذا كان مسجلاً، تعرض له القائمة السفلية التي تتغير تلقائياً لتناسب دوره (معلم أو طالب). 
public class MainActivity extends AppCompatActivity {
    // تبديل المحتوى بناءً على حالة تسجيل الدخول والدور
    private void showMainContent() {
        bottomNav.setVisibility(View.VISIBLE);
        User sessionUser = sessionManager.getCurrentUser();
        if (sessionUser != null && sessionUser.isTeacher()) {
            bottomNav.inflateMenu(R.menu.bottom_nav_teacher);
        } else {
            bottomNav.inflateMenu(R.menu.bottom_nav_student);
        }
        showFragment(new HomeFragment());
    }
}

4. تعديل الملف الشخصي (ProfileEditActivity)

ProfileEditActivity.java
تسمح للمستخدم بتحديث بياناته. تشمل ميزات اختيار صورة من المعرض أو التقاطها بالكاميرا، اختيار التخصصات من قائمة متعددة للمعلمين، وتحديد الصف الدراسي للطلاب. 
public class ProfileEditActivity extends AppCompatActivity {
    private void saveProfile() {
        // تحديث كائن المستخدم في الجلسة وفي قاعدة البيانات السحابية
        userRepository.updateUser(sessionUser, photoUri, result -> {
            if (result.success) {
                sessionManager.saveUserSession(sessionUser);
                finish();
            }
        });
    }
}

6. طبقة إدارة البيانات (Repositories)

هذه الملفات تحتوي على المنطق البرمجي (Logic) للتعامل مع البيانات قبل وبعد إرسالها للسيرفر.
data/UserRepository.java
إدارة بيانات المستخدمين، البحث بالبريد، وتحديث الملف الشخصي مع رفع الصور.
data/LessonRepository.java
إدارة الدروس، طلبات الانضمام، إلغاء الدروس، وجدول المواعيد لكل مستخدم.
data/SessionManager.java
تخزين بيانات الجلسة محلياً باستخدام SharedPreferences لضمان بقاء المستخدم متصلاً.
