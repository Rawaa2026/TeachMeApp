شرح كود شاشة دروس الطلاب (StudentLessonsFragment)

هذه الشاشة هي المسؤولة عن عرض الدروس المتاحة للطالب، حيث يمكنه البحث عن دروس حسب المادة، طلب الانضمام لدرس، أو إلغاء طلب سابق.

1. منطق تحديث البيانات (دالة reload)

تعتبر هذه الدالة "العقل" المفكر لهذه الشاشة. تقوم بعدة عمليات متسلسلة:
  • أولاً: تجلب طلبات الانضمام الخاصة بالطالب الحالي (enrollments) لتعرف حالة كل درس بالنسبة له.
  • ثانياً: تجلب قائمة الدروس القادمة (Upcoming Lessons) مع تطبيق "فلتر" المادة إذا اختاره الطالب.
  • ثالثاً: تقوم بتشغيل خيط (Thread) خلفي لجلب أسماء وأرقام هواتف المدرسين لكل درس لضمان عدم تعليق واجهة المستخدم.

private void reload() {
    progress.setVisibility(View.VISIBLE);
    // 1. جلب التسجيلات الخاصة بي
    repo.listEnrollments(er -> {
        // ... تصفية التسجيلات الخاصة بالطالب

        // 2. جلب الدروس القادمة بناءً على المادة المختارة
        repo.listUpcomingLessons(subjectFilter, lr -> {
            // ... تحديث قائمة الدروس (items)

            // 3. جلب بيانات المدرسين في الخلفية
            new Thread(() -> {
                for (Lesson l : lessonsSnapshot) {
                    User t = userRepo.getUserById(l.teacherId);
                    teacherNames.put(l.teacherId, t.getDisplayName());
                }
                // تحديث الواجهة بعد انتهاء الجلب
                root.post(() -> adapter.notifyDataSetChanged());
            }).start();
        });
    });
}

2. طلب الانضمام لدرس (دالة requestJoin)

عندما يضغط الطالب على زر "طلب انضمام"، يتم إرسال معرف الدرس ومعرف الطالب إلى المستودع (Repository) ليقوم بإنشاء سجل جديد في قاعدة البيانات بحالة "قيد الانتظار" (Pending). 

private void requestJoin(Lesson lesson) {
    repo.requestJoin(lesson.id, u.id, r -> {
        Toast.makeText(getContext(), "تم إرسال طلب الانضمام", Toast.LENGTH_LONG).show();
        reload(); // إعادة تحميل البيانات لتحديث حالة الزر
    });
}

3. منطق عرض الأزرار (في الـ Adapter)

الكود يقوم بتغيير شكل ونصوص الأزرار ديناميكياً بناءً على حالة الدرس:
  • إذا كان الدرس قديماً: يظهر زر "انتهى الدرس" ويكون معطلاً.
  • إذا كان الطلب مقبولاً: يظهر نص "تم التسجيل".
  • إذا كان الطلب قيد الانتظار: يظهر زر "إلغاء الطلب" وزر آخر مكتوب عليه "قيد المراجعة".
  • إذا لم يطلب الطالب بعد: يظهر زر "طلب انضمام" مفعّل.

// منطق تحديد حالة الزر داخل onBindViewHolder
if (LessonEnrollment.STATUS_PENDING.equals(st)) {
    h.action.setText("قيد المراجعة");
    h.action.setEnabled(false);
    h.cancel.setVisibility(View.VISIBLE); // إظهار زر الإلغاء
} else if (LessonEnrollment.STATUS_APPROVED.equals(st)) {
    h.action.setText("تم التسجيل");
    h.action.setEnabled(false);
} else {
    h.action.setText("طلب انضمام");
    h.action.setOnClickListener(v -> requestJoin(l));
}

4. تصفية الدروس حسب المادة (Spinner)

يحتوي الجزء العلوي من الشاشة على قائمة منسدلة (Spinner). عند اختيار مادة معينة، يتم تخزينها في متغير subjectFilter واستدعاء دالة reload() لجلب الدروس المتعلقة بهذه المادة فقط.
