شرح كود مركز المعلم (TeacherHubFragment)

هذا الكلاس يمثل "مركز التحكم" الخاص بالمعلم. بدلاً من وضع الكثير من الأزرار في القائمة السفلية، يستخدم هذا الملف نظام التبويبات (Tabs) لعرض ثلاث صفحات فرعية داخل صفحة واحدة.

1. محول الصفحات (FragmentStateAdapter)

يتم استخدام FragmentStateAdapter لإدارة الصفحات التي تظهر داخل الـ ViewPager2. بناءً على رقم التبويب (position)، يقرر الكود أي "Fragment" سيتم عرضه:
  • التبويب 0: يعرض TeacherLessonsFragment (دروسي المنشورة).
  • التبويب 1: يعرض ScheduleFragment (الجدول الزمني).
  • التبويب 2: يعرض TeacherRequestsFragment (طلبات الطلاب).

pager.setAdapter(new FragmentStateAdapter(this) {
    @NonNull
    @Override
    public Fragment createFragment(int position) {
        switch (position) {
            case 0: return new TeacherLessonsFragment();
            case 1: return new ScheduleFragment();
            case 2: default: return new TeacherRequestsFragment();
        }
    }

    @Override
    public int getItemCount() {
        return 3; // لدينا 3 تبويبات
    }
});

2. ربط التبويبات بالعناوين (TabLayoutMediator)

هذا الجزء هو "الوسيط" الذي يربط الـ TabLayout (الشريط العلوي) مع الـ ViewPager2 (المحتوى المتحرك). وظيفته هي ضبط نصوص العناوين لكل تبويب. 

tabMediator = new TabLayoutMediator(tabs, pager, (tab, position) -> {
    if (position == 0) {
        tab.setText(R.string.nav_my_lessons); // "دروسي"
    } else if (position == 1) {
        tab.setText(R.string.nav_schedule);   // "الجدول"
    } else {
        tab.setText(R.string.nav_join_requests); // "طلبات الانضمام"
    }
});
tabMediator.attach(); // تفعيل الربط

3. تنظيف الذاكرة (onDestroyView)

من المهم جداً في أندرويد فصل الوسيط (Mediator) عند تدمير الواجهة لتجنب حدوث تسريب في الذاكرة (Memory Leak). 

@Override
public void onDestroyView() {
    if (tabMediator != null) {
        tabMediator.detach(); // فصل الربط عند الخروج من الصفحة
        tabMediator = null;
    }
    super.onDestroyView();
}
