شرح كود مدير الجلسات (SessionManager)

يعتبر SessionManager هو المسؤول عن إبقاء المستخدم مسجلاً للدخول حتى بعد إغلاق التطبيق. يقوم بحفظ بيانات المستخدم بشكل دائم على ذاكرة الهاتف باستخدام تقنية SharedPreferences.

1. التهيئة (Initialization)

عند إنشاء نسخة من هذا الكلاس، يتم تجهيز SharedPreferences لتخزين البيانات، ومكتبة Gson لتحويل كائن المستخدم (Object) إلى نص (JSON) وبالعكس. 

public SessionManager(Context context) {
    // PREFS_NAME هو اسم الملف الصغير الذي سيحفظ البيانات في ذاكرة الهاتف
    prefs = context.getApplicationContext().getSharedPreferences("rawaa_session", Context.MODE_PRIVATE);
    gson = new GsonBuilder().create();
}

2. حفظ بيانات الجلسة (saveUserSession)

هذه الدالة تأخذ كائن المستخدم بالكامل، تحوله إلى نص JSON، وتخزنه في ذاكرة الهاتف. كما تقوم بتخزين القيم الأساسية (مثل ID والدور) بشكل منفصل لسهولة الوصول إليها. 

public void saveUserSession(User user) {
    if (user == null) return;

    // تحويل كائن المستخدم إلى نص JSON
    String userJson = gson.toJson(user);

    // حفظ البيانات في الإعدادات
    prefs.edit()
            .putBoolean("logged_in", true) // علامة تدل على أن المستخدم مسجل دخول
            .putString("user_json", userJson)
            .putString("user_id", user.id)
            .putString("user_role", user.role)
            .apply(); // تنفيذ الحفظ
}

3. استرجاع المستخدم الحالي (getCurrentUser)

عندما يحتاج التطبيق لمعرفة من هو المستخدم الحالي، تقوم هذه الدالة بقراءة نص الـ JSON المخزن، وتحويله مجدداً إلى كائن User. كما تحتوي على نظام "تحقق" (Validation) للتأكد من أن البيانات ليست تالفة. 

public User getCurrentUser() {
    if (!isLoggedIn()) return null;

    String userJson = prefs.getString("user_json", null);
    if (userJson != null) {
        // تحويل النص مجدداً إلى كائن جافا
        User user = gson.fromJson(userJson, User.class);
        return validateUser(user); // التأكد من صحة الحقول
    }
    return null;
}

4. دوال التحقق السريع

يوفر الكود دوالاً بسيطة لمعرفة حالة المستخدم بسرعة دون الحاجة لجلب كامل بياناته:
  • isLoggedIn(): هل المستخدم مسجل دخوله حالياً؟
  • isCurrentUserTeacher(): هل المستخدم الحالي معلم؟
  • isCurrentUserStudent(): هل المستخدم الحالي طالب؟

5. تسجيل الخروج (clearSession)

عندما يضغط المستخدم على "تسجيل الخروج"، تقوم هذه الدالة بمسح كافة البيانات المخزنة في الذاكرة المحلية، مما يجعل التطبيق يطلب تسجيل الدخول مرة أخرى في المرة القادمة. 

public void clearSession() {
    prefs.edit()
            .putBoolean("logged_in", false) // إلغاء علامة تسجيل الدخول
            .remove("user_json") // حذف بيانات المستخدم
            .remove("user_id")
            .apply();
}
