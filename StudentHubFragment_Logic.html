شرح كود مركز الطالب (StudentHubFragment)

يعتبر StudentHubFragment هو "المقر الرئيسي" للطالب داخل التطبيق. وظيفته الأساسية هي تنظيم المحتوى الموجه للطالب (مثل تصفح الدروس، البحث عن مدرسين، والجدول الزمني) في مكان واحد باستخدام نظام التبويبات (Tabs).

1. ربط الواجهة (onCreateView)

تقوم هذه الدالة بربط كود الجافا بملف التصميم fragment_role_hub. تم تصميم هذا الملف ليكون مرناً بحيث يحتوي على شريط تبويبات (TabLayout) ومساحة لعرض الصفحات (ViewPager2). 

@Nullable
@Override
public View onCreateView(@NonNull LayoutInflater inflater, @Nullable ViewGroup container,
                         @Nullable Bundle savedInstanceState) {
    // نفخ واجهة الـ Hub الموحدة
    return inflater.inflate(R.layout.fragment_role_hub, container, false);
}

2. إعداد محول الصفحات (FragmentStateAdapter)

داخل دالة onViewCreated، نقوم ببرمجة الـ ViewPager2. هذا المكون هو المسؤول عن السماح للطالب بـ "السحب" (Swipe) بين الصفحات. المحول يحدد أي صفحة تظهر بناءً على الترتيب:
  • الترتيب 0: يفتح StudentLessonsFragment (لتصفح الدروس).
  • الترتيب 1: يفتح StudentTeachersFragment (للبحث عن مدرسين).
  • الترتيب 2: يفتح ScheduleFragment (لعرض الجدول الزمني الشخصي).

pager.setAdapter(new FragmentStateAdapter(this) {
    @NonNull
    @Override
    public Fragment createFragment(int position) {
        switch (position) {
            case 0: return new StudentLessonsFragment();
            case 1: return new StudentTeachersFragment();
            case 2: default: return new ScheduleFragment();
        }
    }

    @Override
    public int getItemCount() {
        return 3; // إجمالي عدد الصفحات في مركز الطالب
    }
});

3. ربط العناوين بالتبويبات (TabLayoutMediator)

هذا الجزء هو "المترجم" الذي يضع الأسماء الصحيحة فوق كل صفحة. يقوم بربط نص العنوان القادم من ملف النصوص (Strings) بالتبويب المقابل له في شريط التبويبات. 

tabMediator = new TabLayoutMediator(tabs, pager, (tab, position) -> {
    if (position == 0) {
        tab.setText(R.string.nav_browse_lessons); // "تصفح الدروس"
    } else if (position == 1) {
        tab.setText(R.string.nav_teachers_by_subject); // "المعلمون"
    } else {
        tab.setText(R.string.nav_schedule); // "الجدول"
    }
});
tabMediator.attach(); // ربط وتفعيل التبويبات مع الصفحات

4. الحفاظ على موارد النظام (onDestroyView)

عندما ينتقل الطالب لشاشة أخرى (مثل الملف الشخصي)، يتم استدعاء هذه الدالة. نقوم هنا بفصل tabMediator لضمان عدم استهلاك الذاكرة بدون داعٍ، مما يحافظ على سرعة التطبيق. 

@Override
public void onDestroyView() {
    if (tabMediator != null) {
        tabMediator.detach(); // فصل الوسيط
        tabMediator = null;
    }
    super.onDestroyView();
}
